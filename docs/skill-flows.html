Skill Overview

4 Skills
/quote

Quote

Read-only. Fetches swap route and displays expected output, rate, and gas cost. No transaction built.

/swap-build

Swap Build

Fetches route, asks for confirmation, then builds encoded calldata. Does NOT submit on-chain.

/swap-execute

Swap Execute

Takes calldata from /swap-build and broadcasts it on-chain via cast send. Requires confirmation.

/swap-execute-fast

Fast Execute

Build + broadcast in one step. Shell script handles everything. Only one verbal confirmation before launch.

/swap-build — Build Swap Transaction

Builds Calldata
Fetches route, shows quote for confirmation, then builds encoded calldata ready for on-chain submission. Does NOT submit. 
flowchart TD
    A([User: swap amount tokenIn to tokenOut
on chain from sender]) --> V

    subgraph VALIDATE ["Input Validation"]
        V[Parse amount, tokenIn, tokenOut,
chain, sender, recipient, slippage]
        V --> V1{sender
provided?}
        V1 -->|No| V2[Ask user for sender]:::warn
        V1 -->|Yes| V3{Valid address?
Not zero/sentinel?}
        V2 --> V3
        V3 -->|Invalid| V4[Reject]:::danger
        V3 -->|Valid| V5[Set slippage defaults
stable=5, common=50, other=100 bps]
    end

    V5 --> B

    subgraph RESOLVE ["Step 1: Resolve Tokens"]
        B[Read token-registry.md] --> C{Found?}
        C -->|Yes| D[Registry address]
        C -->|No| E[Token API → CoinGecko → Ask user]
        E --> D
    end

    D --> F

    subgraph SAFETY ["Step 2: Token Safety"]
        F{Non-registry
token?}
        F -->|Registry| H[Skip]
        F -->|Non-registry| G[GET honeypot-fot-info]
        G --> G1{Honeypot?}
        G1 -->|Yes| G2[REFUSE swap]:::danger
        G1 -->|No| G3{FOT?}
        G3 -->|Yes| G4[Warn about tax]:::warn
        G3 -->|No| H
        G4 -->|User ack| H
    end

    H --> I["Step 3: Convert to wei"]
    I --> J

    subgraph ROUTE ["Step 4: GET Route"]
        J[GET /routes API] --> K{Success?}
        K -->|Error| K1[Handle error code]:::danger
        K -->|OK| L[Extract routeSummary]
    end

    L --> M

    subgraph DUST ["Step 4a: Dust Check"]
        M{Dust or
uneconomical?}
        M -->|Yes| M1[Warn + ask confirmation]:::warn
        M1 -->|User declines| M2[Abort]:::danger
        M1 -->|User accepts| N
        M -->|No| N[OK]
    end

    N --> O

    subgraph CONFIRM ["Step 4b: Quote Confirmation"]
        O[Display quote table:
rate, output, min received,
gas, route, addresses]
        O --> P{User confirms?}
        P -->|No / Cancel| P1[Abort — swap cancelled]:::danger
        P -->|Yes| Q[Proceed to build]
    end

    Q --> R

    subgraph BUILD ["Step 5: POST Build"]
        R["POST /route/build via curl
routeSummary + sender + slippage + deadline"]
        R --> S{code == 0?}
        S -->|4008 stale| S1[Re-fetch route, retry]
        S -->|4227, 4222, etc.| S2[Handle build error]:::danger
        S1 --> R
        S -->|OK| T[Extract tx.data, tx.value, tx.gas]
    end

    T --> U

    subgraph OUTPUT ["Step 6-7: Output"]
        U[Display transaction details:
calldata, value, gas, addresses]
        U --> U1{ERC-20 input?}
        U1 -->|Yes| U2[Show approval reminder]
        U1 -->|No / native| U3[Done]
        U2 --> U3
    end

    U3 --> W(["Calldata ready
Use /swap-execute to submit"])

/swap-execute — Execute Swap On-Chain

Sends Transaction
Takes the JSON output from /swap-build and broadcasts the transaction on-chain using Foundry's cast send. Irreversible.

    subgraph VALIDATE ["Step 1: Validate Input"]
        B{Has tx.to,
tx.data, chain,
sender?}
        B -->|Missing| B1["Ask user to run
/swap-build first"]:::danger
        B -->|Valid| C[Extract tx fields]
    end

    C --> D

    subgraph RPC ["Step 2: Determine RPC"]
        D[Map chain → RPC URL
or use custom --rpc-url]
        D --> D1{RPC

        D1 -->|No| D2[Error: unknown chain]:::danger
        D1 -->|Yes| E[RPC URL ready]
    end

    E --> F

    subgraph CONFIRM ["Step 3: Final Confirmation"]
        F["Display swap summary:
amount, tokens, router,
value, gas, sender"]
        F --> F1{User confirms

        F1 -->|No| F2[Abort]:::danger
        F1 -->|Yes| G[Proceed]
    end

    G --> H

    subgraph SIMULATE ["Step 3b: Simulate (Recommended)"]
        H["cast call --from sender
--value tx.value tx.to tx.data"]
        H --> H1{Reverts?}
        H1 -->|Yes| H2["Warn: would fail on-chain

        H1 -->|No| I[Simulation OK]
    end

    I --> J

    subgraph WALLET ["Step 4: Wallet Method"]
        J{Which signing
method?}
        J -->|Keystore| J1["--account NAME

        J -->|Env var| J2["--private-key $PRIVATE_KEY"]

        J2 --> K
        J3 --> K
        J4 --> K
    end

    subgraph RESULT ["Step 6: Display Result"]

/swap-execute-fast — Build + Execute Immediately

DANGEROUS
Shell script builds the swap and immediately broadcasts. One verbal confirmation, then everything is automated. Includes the new pre-resolution step for unregistered tokens via address:decimals format.

Full Swap Lifecycle — How Skills Connect

Architecture
Two paths to execute a swap: the safe 2-step flow (build → execute) and the fast 1-step flow. Token resolution is shared across all skills. 
flowchart LR
    USER([User Request]) --> DECIDE{What does the
user want?}

    DECIDE -->|"Check price"| QUOTE["/quote
Read-only"]
    DECIDE -->|"Swap with review"| BUILD["/swap-build
Build calldata"]
    DECIDE -->|"Fast swap / immediate"| FAST["/swap-execute-fast
Build + Execute"]

    subgraph SHARED ["Shared: Token Resolution"]
        direction TB
        REG["token-registry.md"]
        API["KyberSwap Token API"]
        CG["CoinGecko API"]
        ASK["Ask User"]
        REG -.->|fallback| API -.->|fallback| CG -.->|fallback| ASK
    end

    QUOTE --> SHARED
    BUILD --> SHARED
    FAST --> SHARED

    QUOTE --> QOUT(["Quote displayed
No transaction"])

    BUILD --> CONFIRM1{"User confirms
quote?"}
    CONFIRM1 -->|Yes| CALLDATA["Encoded calldata
ready"]
    CONFIRM1 -->|No| ABORT1([Cancelled])

    CALLDATA --> EXEC["/swap-execute
Broadcast on-chain"]
    EXEC --> CONFIRM2{"User confirms
execution?"}
    CONFIRM2 -->|Yes| TX1([Transaction on-chain])
    CONFIRM2 -->|No| ABORT2([Cancelled])

    FAST --> CONFIRM3{"Single verbal
confirmation"}
    CONFIRM3 -->|Yes| SCRIPT["Shell script:
resolve → route → build
→ pre-flight → cast send"]
    CONFIRM3 -->|No| ABORT3([Cancelled])
    SCRIPT --> TX2([Transaction on-chain])
