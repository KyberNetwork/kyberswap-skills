## Token Not Listed?

Use this fallback sequence. Stop as soon as you find the token.

### 1. KyberSwap Token API (preferred)

Search by name/symbol using the `name` and `isWhitelisted` parameters. See `references/api-reference.md` for the full spec.

```
# Try whitelisted first (safest)
GET https://token-api.kyberswap.com/api/v1/public/tokens?chainIds={chainId}&name={symbol}&isWhitelisted=true

# If no result, try without isWhitelisted
GET https://token-api.kyberswap.com/api/v1/public/tokens?chainIds={chainId}&name={symbol}
```

The `name` parameter searches both token name and symbol (case-insensitive, partial match). Use `isWhitelisted=true` first to get only KyberSwap-vetted tokens. From the results, pick the one whose `symbol` matches exactly (case-insensitive) with the highest `marketCap`. Extract the `address` and `decimals`.

If the whitelisted search returns no results, retry without `isWhitelisted` but only consider tokens that are `isVerified: true` or have a non-null `marketCap`. If still nothing, fall back to browsing by market cap: query with `page=1&pageSize=100` and scan through pages 1-3.

### 2. CoinGecko API (secondary fallback)

If the Token API doesn't return the token (e.g., very low market cap or newly launched), use CoinGecko's free API to look up verified contract addresses.

**Step 1: Search for the token**

```
GET https://api.coingecko.com/api/v3/search?query={symbol}
```

Via **WebFetch**, fetch this URL and look through the `coins` array in the response. Find the entry whose `symbol` matches the target token (case-insensitive). Note the `id` field.

If multiple results match the symbol, prefer the one with the highest `market_cap_rank` (lowest number = highest rank). If unsure, pick the first exact symbol match.

**Step 2: Get the contract address for the target chain**

```
GET https://api.coingecko.com/api/v3/coins/{id}?localization=false&tickers=false&market_data=false&community_data=false&developer_data=false
```

Via **WebFetch**, fetch this URL using the `id` from Step 1. The response contains a `detail_platforms` object mapping chain names to objects with `contract_address` and `decimal_place`.

**CoinGecko chain name mapping:**

| KyberSwap Chain | CoinGecko Platform Key |
|---|---|
| `ethereum` | `ethereum` |
| `arbitrum` | `arbitrum-one` |
| `polygon` | `polygon-pos` |
| `optimism` | `optimistic-ethereum` |
| `base` | `base` |
| `bsc` | `binance-smart-chain` |
| `avalanche` | `avalanche` |
| `linea` | `linea` |
| `mantle` | `mantle` |
| `sonic` | `sonic` |

Extract the address and decimals from `detail_platforms[chainKey]`:
- `contract_address` — the token's contract address on that chain
- `decimal_place` — the token's decimal precision (e.g., 18, 8, 6)

Example: `detail_platforms["ethereum"] = { "decimal_place": 8, "contract_address": "0x2260fac5e5542a773aa44fbcfedf7c193bc2c599" }` → address is `0x2260...`, decimals is `8`.

If `detail_platforms` is missing or the chain key is not present, fall back to checking the `platforms` object (which only has addresses, no decimals). In this case, you must ask the user to confirm decimals (see below).

**Step 3: Validate the address**

- Verify the address matches the format `^0x[a-fA-F0-9]{40}$` (starts with `0x`, 42 characters total)
- Use the `decimal_place` from `detail_platforms` as the token's decimals
- If `decimal_place` is unavailable, **ask the user to confirm**: "I found the contract address for {TOKEN} on {CHAIN}, but could not verify the decimal precision. Please confirm the number of decimals (common values: 18 for most tokens, 6 for USDC/USDT, 8 for WBTC)."
- **NEVER assume 18 decimals** — an incorrect decimal value will cause the wrong amount to be swapped

### 3. Ask User for Address (final fallback)

If CoinGecko also fails (token not listed, chain not supported, or no contract address found):

- Tell the user: **"I could not find a verified contract address for {TOKEN} on {CHAIN}. Please provide the contract address manually."**
- Once the user provides an address, validate the format (`^0x[a-fA-F0-9]{40}$`)
- Check the honeypot/FOT API before proceeding with the swap
- **NEVER guess or fabricate a token address**
